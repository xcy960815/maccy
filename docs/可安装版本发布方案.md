# 可安装版本发布方案

本文档记录如何把当前 fork 的 Maccy 做成一个可安装、可分发的版本，并支持通过 Homebrew 安装。

## 目标

最终希望用户可以通过以下方式安装你的版本：

```sh
brew install --cask xcy960815/maccy-fork/maccy-xcy
```

或者：

```sh
brew tap xcy960815/maccy-fork
brew install --cask maccy-xcy
```

说明：

- 不建议继续复用官方 `maccy` 这个 Homebrew token。
- 当前 Homebrew 官方 cask `maccy` 指向的是原作者版本，不会自动安装你的 fork。
- 建议为 fork 使用新的 cask 名称，例如 `maccy-xcy`。

## 当前仓库需要先确认的事项

在正式分发前，先明确以下决策：

1. 应用是否要和官方版本并存。
2. 是否保留 Sparkle 自动更新。
3. 第一版是否要求代码签名和公证。
4. Homebrew 安装名称是否确定为 `maccy-xcy`。

建议：

- 如果要和官方版本并存，必须改 `Bundle Identifier`。
- 如果近期不打算维护 Sparkle 更新链路，可以先禁用或暂缓对外使用 Sparkle。
- 如果第一版先追求可安装，可先接受“未签名或未公证”的测试版分发方案。

## 第一步：修改项目内的分发身份

### 1. 修改 Bundle Identifier

当前工程的分发标识已统一调整为 `com.xcy960815.Clipbook`。

需要同步检查的范围：

- `Clipbook` 主程序 target
- `ClipbookTests`
- `ClipbookUITests`
- entitlements 中依赖 `$(PRODUCT_BUNDLE_IDENTIFIER)` 的配置
- 代码里写死的 bundle id、logger label、pasteboard type、`defaults write` 示例

注意：

- logger label 不一定必须改，但如果你想完全区分官方版和 fork 版，建议一起统一。
- pasteboard type 如果继续复用原值，通常也能工作，但从长期维护角度看，建议评估是否改成你自己的命名空间。

### 2. 统一最低系统版本

当前工程最低系统版本与部分发布元数据还没有完全统一。

发布前需要确保以下位置一致：

- Xcode 工程里的 `MACOSX_DEPLOYMENT_TARGET`
- `Info.plist` 中的 `LSMinimumSystemVersion`
- `appcast.xml` 中的 `sparkle:minimumSystemVersion`
- README 中的系统要求说明

建议统一为：

```text
macOS 14
```

## 第二步：准备版本号和发布产物

### 1. 约定版本号策略

不要直接覆盖官方版本号，建议在官方版本上附加自己的后缀，例如：

```text
2.6.17-xcy.1
```

可选方案：

- `2.6.17-xcy.1`
- `2.6.17.1`
- `2.6.17-custom.1`

推荐第一种，最清楚。

### 2. 更新工程版本

发布前同步以下字段：

- `MARKETING_VERSION`
- `CURRENT_PROJECT_VERSION`
- `CFBundleShortVersionString`
- `CFBundleVersion`

### 3. 构建 Release 产物

构建目标：

- 产出一个可分发的 `Maccy.app`
- 再打包为 zip，用于 GitHub Release 和 Homebrew cask

建议命名：

```text
Maccy-2.6.17-xcy.1.app.zip
```

## 第三步：发布到 GitHub Releases

### 1. 创建 GitHub Release

在你的 fork 仓库中创建一个新的 release，并上传 zip 附件。

建议 release tag：

```text
2.6.17-xcy.1
```

建议上传的附件：

- `Maccy-2.6.17-xcy.1.app.zip`

### 2. 校验下载链接

发布后确认最终下载地址可用，例如：

```text
https://github.com/xcy960815/maccy/releases/download/2.6.17-xcy.1/Maccy-2.6.17-xcy.1.app.zip
```

### 3. 计算 SHA256

Homebrew cask 需要 zip 文件的 SHA256。

可使用：

```sh
shasum -a 256 Maccy-2.6.17-xcy.1.app.zip
```

记录输出值，后面写入 cask。

## 第四步：准备 Homebrew Tap

### 1. 新建一个 tap 仓库

建议单独建立一个仓库，例如：

```text
xcy960815/homebrew-maccy-fork
```

原因：

- Homebrew 自定义 cask 最常见的维护方式就是独立 tap 仓库。
- 这样不会和应用源码仓库混在一起。
- 后续升级版本时只需要更新 cask 文件即可。

### 2. 创建 cask 文件

在 tap 仓库里创建：

```text
Casks/maccy-xcy.rb
```

示例内容：

```ruby
cask "maccy-xcy" do
  version "2.6.17-xcy.1"
  sha256 "REPLACE_WITH_REAL_SHA256"

  url "https://github.com/xcy960815/maccy/releases/download/#{version}/Maccy-#{version}.app.zip",
      verified: "github.com/xcy960815/maccy/"
  name "Maccy"
  desc "Clipboard manager"
  homepage "https://github.com/xcy960815/maccy"

  depends_on macos: ">= :sonoma"

  app "Maccy.app"

  uninstall quit: "com.xcy960815.Maccy"

  zap trash: [
    "~/Library/Application Scripts/com.xcy960815.Maccy",
    "~/Library/Containers/com.xcy960815.Maccy",
    "~/Library/Preferences/com.xcy960815.Maccy.plist",
  ]
end
```

注意：

- `version` 要和 GitHub Release tag 对齐。
- `sha256` 要替换成实际值。
- `uninstall` 和 `zap` 中的 bundle id 要和你的应用保持一致。
- `depends_on macos` 要和项目最低系统版本一致。

### 3. 本地验证 cask

可在本机执行：

```sh
brew tap xcy960815/maccy-fork
brew install --cask maccy-xcy
```

如果不想立刻公开，也可以先通过本地 tap 或直接安装本地 cask 文件来验证。

## 第五步：处理 Sparkle 自动更新

当前仓库已经接入 Sparkle，并配置了自己的 `appcast.xml` 和 `SUFeedURL`。

发布前要做二选一：

### 方案 A：继续维护 Sparkle

需要保证以下内容完整可用：

- `appcast.xml` 指向你自己的 release
- `SUFeedURL` 指向你自己的 `appcast.xml`
- `sparkle:minimumSystemVersion` 和实际工程一致
- 每次发布时同步更新 appcast 内容
- 后续补齐 Sparkle 签名链路

适合：

- 你准备长期维护自动更新体验

### 方案 B：暂时不对外使用 Sparkle

如果短期内只是想先把安装链路打通，可以先不依赖 Sparkle 自动更新。

适合：

- 先验证 fork 版本是否稳定
- 先只走 GitHub Release + Homebrew 更新

## 第六步：签名与公证

这一步不是“能安装”的最低要求，但对正式分发非常重要。

### 如果不做签名和公证

可能出现：

- macOS Gatekeeper 安全提示
- 用户首次启动门槛更高
- 安装体验不够顺滑

### 如果要做正式分发

建议后续补齐：

1. Apple Developer 证书
2. App 签名
3. Hardened Runtime 检查
4. notarization 公证
5. stapler 绑定公证票据

## 推荐的最小可行方案

如果目标是尽快让别人装上你的 fork，建议按下面顺序执行：

1. 修改 `Bundle Identifier`
2. 统一最低系统版本和版本号
3. 构建 `Release` 的 `Maccy.app`
4. 压缩为 zip 并创建 GitHub Release
5. 新建 tap 仓库并添加 `maccy-xcy.rb`
6. 本地用 `brew install --cask` 验证安装

这样就已经能形成一条可用的安装链路。

- 自动计算 SHA256 并更新 tap 仓库
